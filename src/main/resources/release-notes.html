v4.0.5

2026-03-18

Bug Fixes

  • Fixed TT unavailable boundary — trucks are now recoverable (reset + reallocate) through TT_DRIVE_UNDER_QC; cancellation only triggers once TT_HANDOVER_FROM_QC is active or completed

v4.0.4

2026-03-18

Improvements

  • ActionStatus moved from ScheduleRunnerProcessor inner enum to a domain-level enum on the Action record — action status is now a first-class field (action.status()) instead of tracked externally via separate sets
  • Canceled containers no longer trigger spurious reschedules — when a container's actions are force-completed (e.g., TT unavailable), its QC Discharged event is ignored and the non-canceled twin's discharge is correctly recognized as the expected next WI

Internal

  • Removed activeActionIds and completedActionIds sets from ScheduleState; all status tracking now lives on the Action record via withStatus()
  • New WorkInstructionCanceled side effect (propagating event) informs WorkQueueProcessor when work instructions are canceled, preventing false order-mismatch reschedules

v4.0.3

2026-03-18

New Features

  • TT unavailable event handling — when a terminal truck becomes unavailable mid-operation, the system either reassigns work or force-completes remaining actions depending on workflow progress
  • Sub-processor pattern (ScheduleSubProcessor) for encapsulating schedule event handling logic, with controlled state access via ScheduleContext
  • TruckUnassigned side effect emitted when a truck is removed from actions
  • CompletionReason enum and completionReason field on actions and ActionCompleted side effect to track force-completion causes

Improvements

  • Schedule viewer shows force-completed actions with gray background and ⚠ icon instead of the normal green check mark
  • Before TT-under-QC activation: truck is unassigned from all TT actions and a new truck is allocated if available
  • After TT-under-QC activation: all remaining actions for the container and its twin are completed with reason TT_UNAVAILABLE

v4.0.2

2026-03-18

New Features

  • Trucks page now shows an "Assignment" column displaying which WQ / container / action each truck is currently assigned to

Improvements

  • Assignment column shows pending actions (not just active ones), matching the engine's truck reservation logic
  • Pending assignments are labeled with (pending) to distinguish from active ones
  • Assignment column updates in real-time via SSE
